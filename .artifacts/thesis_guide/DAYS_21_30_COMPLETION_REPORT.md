## Key Features of All Step Files

### Standard Structure

Every step file follows this template:
1. **Header**: Time estimate, output description, source files
2. **OBJECTIVE**: Clear task description
3. **SOURCE MATERIALS TO READ FIRST**: Preparation with time allocation
4. **EXACT PROMPT TO USE**: Ready-to-copy AI prompt with full specs
5. **WHAT TO DO WITH THE OUTPUT**: Review and formatting procedures
6. **VALIDATION CHECKLIST**: Quality gates before proceeding
7. **EXPECTED OUTPUT SAMPLE**: Example LaTeX/text
8. **COMMON ISSUES**: Troubleshooting guide
9. **TIME CHECK**: Breakdown verification
10. **NEXT STEP**: Explicit pointer to next file

### Quality Standards Enforced

All prompts enforce:
- NO conversational language ("Let's explore", "We can see")
- NO vague claims ("comprehensive" without quantification)
- YES specific statements with citations
- YES technical precision
- Proper LaTeX formatting
- IEEE citation style
- Mathematical rigor

### Integration Points

Step files integrate with:
- **Automation scripts**: `extract_bibtex.py`, `generate_figures.py`, `csv_to_table.py`, `build.sh`
- **LaTeX templates**: Chapter, theorem, figure, table, appendix templates
- **Project docs**: `docs/theory/`, `src/controllers/`, `config.yaml`
- **Experimental data**: `optimization_results/`, `benchmarks/`
